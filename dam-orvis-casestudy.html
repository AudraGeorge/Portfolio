PORTFOLIO PIECE 1: ORVIS CASE STUDY

Content Library Audit & Organizational Redesign

Client: Orvis (National Direct-Mail Catalog Retailer) Role: Digital Asset Management Specialist Timeline: May 2023 – December 2024 Tools: Adobe Experience Manager (AEM), Excel, Google Sheets

PROJECT OVERVIEW

Background

Orvis is a national direct-mail catalog retailer specializing in outdoor lifestyle products. The creative team manages thousands of product images, lifestyle photography, and marketing assets to support seasonal catalogs, e-commerce, and promotional campaigns.

Challenge

Over several years of rapid catalog production, the Adobe Experience Manager (AEM) digital asset library had become disorganized and difficult to navigate. Assets were inconsistently named, metadata was incomplete or missing, and duplicate files cluttered the system. The creative and marketing teams were spending excessive time searching for assets, often re-creating files that already existed because they couldn't locate them.

Key Pain Points:
  • No standardized folder structure (assets scattered across ad-hoc folders)
  • Inconsistent file naming conventions (multiple formats in use)
  • 47% of assets missing critical metadata (product category, season, usage rights)
  • Estimated 15% duplicate files consuming storage and creating version confusion
  • Search functionality severely limited by poor metadata and inconsistent naming
  • New team members took 3-4 weeks to learn the system due to lack of structure

    • Objective

    Conduct a comprehensive audit of the existing digital asset library, identify inefficiencies, and implement a scalable organizational system that would:

  • Reduce time spent searching for assets
  • Eliminate duplicate files
  • Improve metadata consistency and searchability
  • Establish clear governance standards for ongoing maintenance
  • Enable faster onboarding of new team members

    • AUDIT PROCESS & FINDINGS

    Methodology

    Phase 1: Inventory & Assessment (Weeks 1-3)
  • Cataloged all existing folders and subfolders in AEM
  • Analyzed file naming patterns across 10,000+ assets
  • Assessed metadata completeness (required fields vs. actual population)
  • Identified duplicate files using hash comparison and visual review
  • Surveyed creative and marketing teams (12 stakeholders) on pain points and workflow needs
    • Phase 2: Data Analysis (Week 4)
  • Categorized findings by severity and impact on daily operations
  • Mapped user workflows to identify most critical use cases
  • Benchmarked against DAM industry best practices
  • Prioritized issues based on frequency of occurrence and time wasted

    • Key Findings

    1. Folder Structure Chaos
  • Finding: 78 top-level folders with no clear organizational logic
  • Examples of Problems:

    • - "Spring 2023," "2023 Spring," "Spring23" all existed as separate folders

    - Assets for same product line scattered across multiple locations

    - No separation between final assets, raw files, and work-in-progress

  • Impact: Users averaging 8-12 minutes to locate a single asset
    • 2. Inconsistent File Naming
  • Finding: 6 different naming patterns in use across the library
  • Examples:

    • - IMG_0034.jpg (camera default, no context)

    - orvis_fly_rod_lifestyle_hero.jpg (descriptive but no version/date)

    - FlyRod_Spring2023_v2_FINAL.jpg (includes version but inconsistent format)

    - 23SPR_ProductShot_001.jpg (abbreviated, cryptic)

  • Impact: Filename searches frequently returned no results; users relied on visual browsing
    • 3. Metadata Gaps
  • Finding: 47% of assets missing critical metadata fields
  • Missing Data Breakdown:

    • - Product category: 52% missing

    - Season/campaign: 41% missing

    - Usage rights/licensing: 68% missing

    - Photographer/creator credit: 73% missing

  • Impact: Advanced search functionality rendered nearly useless; legal risk from unclear usage rights
    • 4. Duplicate Files
  • Finding: Approximately 1,500 duplicate assets (15% of total library)
  • Causes:

    • - Same asset uploaded to multiple folders for "easier access"

    - Renamed files treated as new assets

    - Lack of version control led to saving near-identical variants

  • Impact: Wasted storage costs; confusion over which version was current/approved
    • 5. Lack of Governance
  • Finding: No documented standards or training materials for DAM usage
  • Consequences:

    • - Each team member used personal organizational logic

    - No review process for new uploads

    - No scheduled maintenance or cleanup

  • Impact: System degradation accelerating over time; problems compounding

    Design Principles:
  • Numbered prefixes enforce logical order and prevent alphabetical sorting chaos
  • Product-first organization aligns with team workflows (search by product type, then season)
  • Consistent season folders across all product categories
  • Clear separation of final assets vs. brand resources vs. archived content
  • Archive at bottom (99 prefix) to keep out of daily navigation

    • File Naming Conventions

    Established a single, mandatory naming standard:

    Format: 
    [ProductCategory]_[ProductName]_[ImageType]_[Season]_[Version]_[Date].extension
    Examples: 
    FlyFishing_EvoRod_ProductShot_2024Spring_v01_20240115.jpg

    Apparel_TroutBumHat_Lifestyle_2024Spring_v02_20240118.jpg
    

    Marketing_SpringCatalog_EmailHero_2024Spring_v01_20240120.jpg
    Naming Rules:
  • No spaces (use underscores for readability)
  • CamelCase for multi-word product names (EvoRod, TroutBumHat)
  • Consistent season format: YYYYSeason (2024Spring, 2024Fall)
  • Version format: v## (v01, v02)
  • Date format: YYYYMMDD (20240115)
  • Maximum length: 100 characters
    • Benefits:
  • Files self-describe content without opening
  • Alphabetical sorting groups related assets
  • Version and date enable tracking iterations
  • Search by any component (product name, season, image type)

    • Metadata Schema

    Defined required and optional metadata fields for all assets:

    FieldTypeRequiredPurposeExamples
    TitleTextYesDescriptive asset name"Evo Fly Rod - Product Shot"
    Product CategoryDropdownYesPrimary classificationFly Fishing, Apparel, Home Goods
    Product NameTextYesSpecific productEvo Rod, Trout Bum Hat
    Image TypeDropdownYesUsage contextProduct Shot, Lifestyle, Detail, Swatch
    Season/CampaignDropdownYesAssociated release2024 Spring, 2024 Fall, Holiday 2023
    KeywordsTagsYesSearchable termsoutdoor, fishing, river, men's, women's
    Usage RightsDropdownYesLicensing statusFull Rights, Limited Use, Stock License
    PhotographerTextNoCreator creditName or agency
    Shot LocationTextNoGeographic contextVermont, Montana, Studio
    File StatusDropdownYesWorkflow stateFinal/Approved, Draft, Archive
    Mandatory Fields Enforcement:
  • Assets cannot be uploaded without completing required fields
  • Dropdown menus prevent free-text inconsistencies
  • Metadata entry integrated into upload workflow (not retroactive burden)

    • Governance & Maintenance Standards

    Created documentation to prevent system degradation over time:

    Upload Standards

  • All new assets must follow folder structure and naming conventions
  • Metadata completion mandatory before upload
  • Duplicate check required (search existing assets before uploading new)

    • Review & Approval Process

  • Creative lead reviews new uploads weekly
  • Flagged items (missing metadata, unclear naming) corrected within 48 hours

    • Quarterly Audits

  • Review metadata completeness across random sample (200 assets)
  • Identify and merge duplicate files
  • Archive assets older than 24 months with no recent usage

    • Training & Onboarding

  • New team members complete DAM training within first week
  • Quick reference guide posted in shared drive
  • "DAM Champion" role assigned to answer questions and maintain standards
    • Governance Document Includes:
  • Folder structure map
  • Naming convention examples
  • Metadata field definitions
  • Upload checklist
  • Troubleshooting FAQ

    • IMPLEMENTATION & OUTCOMES

    Implementation Approach

    Phase 1: Pilot Restructure (Weeks 5-6)
  • Reorganized one product category (Fly Fishing) as proof of concept
  • Migrated 800 assets to new structure with corrected metadata
  • Solicited feedback from creative team before scaling
    • Phase 2: Full Library Migration (Weeks 7-12)
  • Systematically moved remaining 9,200+ assets to new folder structure
  • Bulk metadata updates where possible (season, product category)
  • Manual review of high-value assets for accuracy
    • Phase 3: Duplicate Removal (Weeks 13-14)
  • Identified and consolidated duplicate files
  • Preserved highest-resolution version; deleted lower-quality copies
  • Archived near-duplicates (slight variations) with clear version notes
    • Phase 4: Training & Rollout (Week 15)
  • Conducted live training sessions (3 sessions for global team coverage)
  • Distributed governance documentation and quick reference guides
  • Designated "DAM Champions" in each department for ongoing support
    • Phase 5: Monitoring & Refinement (Weeks 16-18)
  • Weekly check-ins with team to address confusion or issues
  • Minor adjustments to metadata dropdowns based on user feedback
  • Established quarterly audit schedule

    • Outcomes & Impact

    Efficiency Gains:
  • Search time reduced by 40%: Average asset retrieval time dropped from 8 minutes to 5 minutes
  • Duplicate files eliminated: Removed 1,500 duplicate assets, freeing storage and reducing confusion
  • Metadata completion increased to 98%: Nearly all assets now fully searchable
  • Onboarding time cut in half: New team members productive in 1-2 weeks vs. 3-4 weeks
    • Operational Improvements:
  • Version control clarity: Clear naming and metadata eliminated "which file is final?" questions
  • Legal compliance: Usage rights field reduced risk of unauthorized asset use
  • Cross-team collaboration: Marketing and creative teams could find shared assets without asking colleagues
  • Scalability: Structure easily accommodates new product lines and seasonal campaigns
    • User Satisfaction:
  • Post-implementation survey: 92% of users rated new system as "significantly better" or "much easier to use"
  • Reduced Slack messages asking "where is the file for...?" by an estimated 70%
    • Long-Term Sustainability:
  • Governance standards adopted by team; upload quality maintained post-implementation
  • Quarterly audits identify small issues before they compound
  • System remained organized 18+ months post-launch with minimal intervention

    Governance Document Excerpt

    DAM Upload Checklist:

    Before uploading any asset to Adobe Experience Manager:

  • [ ] File follows naming convention: [Category]_[ProductName]_[ImageType]_[Season]_[Version]_[Date].extension
  • [ ] File is saved in correct folder based on product category and season
  • [ ] All required metadata fields completed (Product Category, Season, Usage Rights, File Status)
  • [ ] Asset is final/approved version (not draft or work-in-progress)
  • [ ] Searched existing library to confirm asset doesn't already exist (avoid duplicates)
  • [ ] File is high-resolution and export-ready (no further edits needed)
    • If unsure about any step, contact the DAM Champion before uploading.

    CONCLUSION

    This project transformed a chaotic, difficult-to-navigate digital asset library into a streamlined, searchable system that saved the creative team hours per week and reduced operational friction. By establishing clear organizational principles, enforcing metadata standards, and creating governance documentation, the redesigned library became a sustainable asset that scaled with the business.

    Key Success Factors:
  • User-centered design: structure aligned with actual team workflows
  • Comprehensive audit before implementation: understood root causes, not just symptoms
  • Clear governance: prevented backsliding into old habits
  • Training and support: ensured adoption and correct usage

The reorganized DAM system continues to support Orvis's seasonal catalog production and marketing campaigns with improved efficiency and reduced time-to-asset retrieval.
