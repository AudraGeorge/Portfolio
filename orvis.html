Case Study

Orvis

Resurrecting a Legacy DAM

15%

Improvement in retrieval efficiency

The Problem

The company's asset and intake systems were set up years ago and left without governance. This "oral tradition" created a chaotic "messy closet" that resulted in daily, pointless work and constant burnout.

The Solution & My Work

I implemented a systematic solution to audit the mess, build a new structure, and document the standards that had been missing.

Information Architecture & Audit

The first step was to diagnose the problem. I conducted a full audit of the AEM digital asset catalogs, removing duplicates and implementing consistent file naming conventions to create a logical foundation.

Taxonomy & Metadata

With a clean foundation, I built the system's "language." I developed standardized metadata schemas and taxonomies across 10,000+ assets so users could filter instead of guessing.

Workflow & Process

To stop the mess at the source, I rebuilt the broken intake system in Workfront. I developed new intake forms that forced all new assets to have clean metadata *before* entering the DAM.

Artifact: New Intake Form

Governance & Standards

To make the fix *permanent*, I solved the "nothing is written down" problem by creating the clear SOPs (Standard Operating Procedures) and training documentation that had been missing.

Artifact: Standard Operating Procedure (SOP) Snippet

STANDARD OPERATING PROCEDURE

File Naming Convention

01
Use the Standard Format

All files must follow the YYYY-MM-DD_[Project]_[Description]_[v#] format.

02
No Spaces Allowed

Use underscores (_) or hyphens (-) to separate words. Do not use spaces.

03
Version Control is Mandatory

Always update the version number (e.g., _v1, _v2) when saving a new iteration of a file.
