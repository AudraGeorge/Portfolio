Case Study (DAM Remediation)

Orvis

Resurrecting a Legacy DAM

15%

Faster Asset Retrieval

The Problem

The company's DAM system was set up years ago and then abandoned‚Äîno governance, no standards, no one responsible for upkeep. Over time, it became a digital junk drawer where assets went to die.

Teams spent hours searching for files that might not exist, or recreating assets that did exist but couldn't be found. The constant friction was burning people out.

The Solution & My Work

I rebuilt the system from the ground up: audited 10,000+ assets, established information architecture and taxonomy standards, redesigned intake workflows, and documented everything so it would stay functional.

Information Architecture & Audit

The first step was diagnosing the damage. I audited the entire AEM library‚Äî10,000+ assets with inconsistent naming, duplicates, and no discernible structure.

The Challenge

Files were named whatever the uploader felt like that day: "Final_FINAL_v2.jpg," "Copy of H4_Ad.jpg," "random_stuff_001.mov." Searching for anything meant wading through dozens of near-identical filenames. No one knew which version was approved or where the original lived.

I removed duplicates, standardized file names with date-product-version conventions, and rebuilt the folder structure around product lines and asset types.

Taxonomy & Metadata Standards

Clean file names helped, but didn't solve search. I developed a metadata schema with controlled vocabularies‚Äîstandardized tags for product line, asset type, and approval status‚Äîso people could filter instead of guess.

The Challenge

The old system had freeform text fields. People tagged assets with whatever came to mind: "cool photo," "approved maybe?," "H4 stuff." Searches returned everything or nothing. I needed a taxonomy rigid enough to enable consistent search, but flexible enough that people would actually use it.

I mapped 10,000+ assets to standardized metadata fields with dropdown menus and controlled values‚Äîno more guessing.

Intake Workflow Redesign

The DAM could stay clean only if new assets entered correctly. The old intake system in Workfront was optional and incomplete‚Äîmost people skipped it entirely and uploaded files directly, with no metadata.

I rebuilt the intake form to enforce the standards. Required fields, dropdown menus, and validation rules meant assets couldn't enter the DAM dirty.

Artifact: Redesigned Intake Form

Documented Governance

The original system failed because nothing was written down‚Äîit only existed in people's heads. I created SOPs for file naming, metadata tagging, and intake processes so new team members could onboard without tribal knowledge.

Artifact: SOP Snippet

STANDARD OPERATING PROCEDURE

File Naming Convention

01
Use the Standard Format

All files must follow the YYYY-MM-DD_[Product]_[Description]_[v#] format.

02
No Spaces Allowed

Use underscores (_) to separate words. Do not use spaces.

03
Version Control is Mandatory

Always update the version number (e.g., _v1, _v2) when saving iterations.

The Payoff

The 15% retrieval improvement meant teams stopped wasting hours hunting for files. Duplicate asset creation dropped. The system became the trusted source of truth it was supposed to be from the start‚Äîand it stayed that way because the governance was finally documented.
