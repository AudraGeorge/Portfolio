DAM PORTFOLIO PIECE 2: GOVERNANCE FRAMEWORK

Digital Asset Governance Framework

Deloitte Global Creative Operations Implementation Period: 2018‚Äì2022

Executive Summary

This governance framework established standards for managing creative assets‚Äîphotography, templates, brand materials, video, and marketing collateral‚Äîacross Deloitte's global creative operations team. As the team expanded internationally to serve 40+ members across multiple regions and time zones, a centralized system with clear governance became critical to maintaining asset quality, ensuring brand consistency, and enabling efficient retrieval.

The framework increased metadata completeness from 32% to 96%, reduced asset search time by 65%, and eliminated over 7,500 duplicate files, creating a sustainable digital asset ecosystem that scaled globally.

Purpose & Scope

Purpose

To establish clear standards for uploading, organizing, maintaining, and accessing creative assets stored in Adobe Experience Manager (AEM), ensuring consistency, searchability, and brand compliance across global teams.

Scope

This framework applies to:

  • Photography (corporate, lifestyle, industry-specific, stock)
  • Templates (presentation, design, document)
  • Brand assets (logos, style guides, color palettes, typography)
  • Marketing materials (campaign assets, collateral, event materials)
  • Video assets (brand videos, case studies, stock footage)
    • Out of Scope: Internal documentation, process SOPs, and training materials (managed separately through knowledge base).

    Asset Upload Standards

    Required Pre-Upload Checks

    Before uploading any asset to Adobe Experience Manager:

  • Duplicate Check

    • - Search AEM using keywords to verify asset doesn't already exist

    - If similar asset exists, evaluate whether new upload is necessary or if existing asset can be reused

    - Reduces storage costs and prevents library clutter

  • Quality Verification

    • - Photography: Minimum 300 DPI for print use; RGB color mode for digital

    - Templates: Tested and functional; all fonts embedded or outlined; links updated

    - Video: Minimum 1080p resolution; appropriate file format (MP4 for web, MOV for editing)

    - Brand assets: Vector format where applicable; multiple file formats provided (AI, EPS, PNG)

  • Version Validation

    • - Templates must be marked with version number (v01, v02, etc.)

    - Current version clearly identified via metadata "File Status" field

    - Older versions moved to Archive folder but retained for reference

  • Brand Compliance

    • - Logos and brand assets reviewed by Brand Team before upload

    - Templates checked against current brand guidelines

    - Outdated brand materials archived and replaced

    File Naming Conventions

    Standard Format

    All assets must follow this naming convention:

    [AssetType]_[Category]_[Description]_[Region]_[Version]_[Date].extension

    Naming Rules

  • No spaces: Use underscores for readability
  • Asset type prefix required: Photo, Template, Logo, Campaign, Video, etc.
  • Description concise but clear: Max 3-4 words describing content
  • Region tag mandatory: Global, EMEA, APAC, Americas, or specific country code
  • Version format: v## with leading zero (v01, v02, v10)
  • Date format: YYYYMMDD (20230415)
  • Maximum filename length: 100 characters
  • Case sensitivity: Use consistent capitalization (CamelCase for multi-word descriptions)

    Folder Organization Rules

  • Assets placed in only one location: No cross-posting to multiple folders
  • Campaign assets nested by campaign name within Marketing_Materials folder
  • Regional variations nested within asset type (not separate regional folders at top level)
  • Archive folder for retired assets: Maintains clean active directories
  • Maximum folder depth: 3 levels to prevent deep nesting and "click fatigue"

    • Metadata Schema & Requirements

    Required Metadata Fields

    All assets must have the following metadata completed before upload:

    FieldTypePurposeExamples
    Asset TypeDropdownPrimary classificationPhotography, Template, Logo, Brand Asset, Video
    CategoryDropdownSecondary classificationLifestyle, Portrait, Corporate, Industry
    DescriptionText (50 char max)Concise asset description"Team collaboration in modern office"
    Region/MarketMulti-selectGeographic applicabilityGlobal, EMEA, APAC, Americas, [Country]
    File StatusDropdownVersion statusCurrent, Archive, Draft
    Usage RightsDropdownLicensing statusFull Rights, Stock License, Limited Use, Internal Only
    KeywordsTags (min 3)Search termsoffice, collaboration, team, professional, modern

    Optional Metadata Fields

    Strongly encouraged for improved searchability:

    FieldTypePurposeExamples
    Campaign/ProjectDropdownAssociated initiativeQ2 Launch, Annual Report 2023, [Client Name]
    IndustryMulti-selectIndustry focusFinancial Services, Technology, Healthcare, Government
    Creator/PhotographerTextAttribution[Name or Agency]
    OrientationDropdownLayout orientationHorizontal, Vertical, Square
    Color PaletteTagsDominant colorsBlue, Green, Neutral, Warm Tones
    Model ReleaseCheckboxLegal release statusYes/No

    Metadata Enforcement

  • Required fields enforced via AEM workflow: Cannot complete upload without populating mandatory metadata
  • Dropdown menus prevent inconsistencies: Standardized values eliminate free-text variations
  • Multi-select fields enable faceted search: Users can filter by multiple criteria simultaneously
  • Metadata templates for bulk uploads: Expedite large-scale migrations while maintaining standards

    • Asset Lifecycle Management

    1. Asset Creation & Upload

    Workflow:
  • Asset finalized and approved by creative lead
  • Uploader completes pre-upload checklist (duplicate check, quality verification, brand compliance)
  • File named according to convention
  • Asset uploaded to appropriate AEM folder
  • All required metadata fields completed
  • Upload submitted; AEM workflow validates compliance
    • Timeline: Standard assets available in AEM within 24 hours of approval

    2. Review & Approval Process

    Brand Assets (Logos, Guidelines):
  • Uploaded to draft folder
  • Routed to Brand Team for approval via AEM workflow
  • Approved assets moved to live folder and marked "Current"
  • Unapproved assets flagged for revision
    • Templates (Presentation, Design, Document):
  • Creative lead reviews for brand compliance
  • Tested for functionality (fonts, links, layout)
  • Approved templates added to "Current Templates" collection
  • Previous version moved to Archive if replaced
    • Photography & Marketing Materials:
  • Creative lead spot-checks for quality and metadata completeness
  • No formal approval required for photography unless brand-sensitive

    • 3. Maintenance & Update Cycles

    Asset TypeReview FrequencyTrigger for Ad-Hoc Update
    PhotographyAnnual archive reviewNew brand guidelines; outdated content
    TemplatesSemi-annualRebrand; software version updates; legal changes
    Brand AssetsQuarterlyBrand refresh; new logo variations; legal updates
    Marketing MaterialsQuarterlyCampaign completion; outdated collateral
    Video AssetsAnnualRebrand; outdated messaging

    4. Archiving Protocol

    When to Archive:
  • Asset replaced by newer version (templates, brand guidelines)
  • Campaign concluded and materials no longer in active use (marketing materials)
  • Outdated brand standards superseded by rebrand (logos, guidelines)
  • Asset unused for 18+ months (photography, video)
    • Archive Process:
  • Update metadata "File Status" to "Archive"
  • Move to 99_Archive folder with appropriate year/quarter subfolder
  • Retain for 2 years before permanent deletion (legal/historical reference)
  • Update any collections or links to remove archived assets

    • Quality Assurance

    Pre-Publication Checklist

    Before publishing any asset to Adobe Experience Manager, verify:

  • [ ] File Quality

    • - [ ] Resolution meets standards (300 DPI for print, 1080p for video)

    - [ ] Color mode correct (RGB for digital, CMYK for print)

    - [ ] File format appropriate for intended use

  • [ ] Naming Convention

    • - [ ] Follows standard format: [AssetType]_[Category]_[Description]_[Region]_[Version]_[Date].ext

    - [ ] No spaces (underscores used)

    - [ ] Version number included

    - [ ] Date in YYYYMMDD format

  • [ ] Folder Location

    • - [ ] Saved in correct folder based on asset type and category

    - [ ] Not duplicated across multiple folders

  • [ ] Metadata

    • - [ ] All required fields completed (Asset Type, Region, File Status, Usage Rights, Keywords)

    - [ ] Description concise and accurate

    - [ ] Minimum 3 keywords tagged

  • [ ] Version Control (Templates only)

    • - [ ] Version number updated from previous version

    - [ ] If "Current," marked as such in File Status

    - [ ] Previous version moved to Archive if replaced

  • [ ] Duplicate Check

    • - [ ] Searched AEM to confirm asset doesn't already exist

    - [ ] If similar asset found, evaluated whether new upload necessary

  • [ ] Brand Compliance (Brand Assets & Templates)

    • - [ ] Reviewed against current brand guidelines

    - [ ] Approval obtained from Brand Team or creative lead

    Quarterly Audit Process

    Conducted by: Creative Operations Specialist + Regional Asset Champions Frequency: Every 3 months (Q1, Q2, Q3, Q4) Scope: All active assets in AEM Audit Checklist:
  • Metadata Completeness

    • - Review random sample of 500 assets

    - Check required fields populated

    - Flag assets with missing metadata for correction

  • Folder Structure Adherence

    • - Verify assets in correct folders

    - Identify misplaced assets and correct

  • Duplicate Detection

    • - Run hash comparison to identify exact duplicates

    - Visual review of near-duplicates

    - Consolidate duplicates (keep highest quality, archive or delete lower quality)

  • Version Control

    • - Review "Current Templates" collection for accuracy

    - Ensure outdated template versions moved to Archive

    - Verify version numbers incremented correctly

  • Archiving Review

    • - Identify assets unused for 18+ months (photography, video)

    - Identify concluded campaigns with active marketing materials

    - Move candidates to Archive folder

  • User Feedback

    • - Solicit team input on asset organization and searchability

    - Identify gaps or confusing areas

    - Adjust metadata dropdowns or folder structure if needed

    Audit Documentation:
  • Findings logged in Audit Report (Excel)
  • Action items assigned to responsible parties
  • Follow-up review 30 days post-audit to verify corrections

    • Access Controls & Permissions

    Permission Levels

    User GroupAccess LevelPermissions
    Global Creative TeamView/DownloadView and download all assets; cannot upload or edit metadata
    Creative Team Members (Contributors)Upload/EditUpload new assets; edit metadata on own uploads; download all assets
    Creative Team LeadsReview/ApproveApprove templates and brand assets; edit metadata on all assets; archive outdated materials
    Creative Operations SpecialistFull ControlAll permissions; folder structure management; metadata schema updates; user permission management
    Brand TeamReview/Approve (Brand Assets)Approve logos, guidelines, and brand-related templates; edit brand asset metadata
    Regional Asset ChampionsUpload/Edit (Regional Scope)Upload and edit assets for their region; support regional teams; participate in audits

    Access Management

  • New team members: Granted "View/Download" access upon onboarding; upgraded to "Upload/Edit" after completing DAM training
  • Departing team members: Access revoked within 24 hours of departure
  • External vendors/freelancers: Temporary "View/Download" access for specific folders; revoked upon project completion
  • Permissions reviewed quarterly: Adjusted based on role changes or team restructuring

    • Training & Support

    Onboarding Training

    All new creative team members complete DAM training within first 2 weeks:

    Training Components:
  • Video tutorial: "Navigating the AEM Asset Library" (15 minutes)
  • Quick reference guide: File naming conventions and metadata requirements (1-page PDF)
  • Hands-on practice: Upload 2-3 sample assets with guidance
  • Q&A session: 30-minute live session with Creative Operations Specialist or Regional Asset Champion
    • Training Resources:
  • AEM Asset Upload Checklist (laminated card for desk reference)
  • Metadata Field Definitions Guide
  • "Common Mistakes & How to Avoid Them" document

    • Ongoing Support

    First-Line Support:
  • Regional Asset Champions field questions from team members in their region
  • Slack channel: #creative-asset-library for quick questions
    • Escalation:
  • Complex issues or system-level questions escalated to Creative Operations Specialist
  • Brand compliance questions routed to Brand Team
    • Refresher Training:
  • Annual refresher webinar for all team members (optional but encouraged)
  • Mandatory refresher if audit reveals repeated non-compliance

    • Maintenance Schedule

    Weekly Tasks

  • Monitor new asset uploads for quality and compliance
  • Address urgent metadata corrections flagged by team
  • Respond to support questions in #creative-asset-library Slack channel

    • Monthly Tasks

  • Review "Current Templates" collection for accuracy
  • Update metadata dropdown values if new categories needed
  • Analyze asset download metrics to identify most-used assets

    • Quarterly Tasks

  • Conduct comprehensive asset library audit
  • Review and update folder structure as needed (rare)
  • Solicit team feedback via survey
  • Update training materials if governance changes made

    • Annual Tasks

  • Comprehensive governance framework review
  • Assess metadata schema for gaps or needed additions
  • Archive photography and video assets unused for 18+ months
  • Review access permissions and adjust as needed
